# journal/ — per-agent session journals

The hybrid model's second half (INDEX.md contract #4): **shared sheets hold
durable facts; journals hold each agent's own experience** — what it built,
tried, suspected, and would do differently. Different models notice different
things; keeping their voices separate preserves that instead of blending it
into mush.

## Convention

- One file per agent: `claude.md`, `codex.md`, `gemini.md`, …
- Newest entry at top, dated, with the session's repo + branch.
- Write in first person. Suspicions and dead ends are WELCOME here (they're
  noise in LEARNINGS, gold in a journal).
- An agent reads its OWN journal on session start for continuity, and may
  read others' for perspective — but never edits another agent's journal.
- When a journal suspicion gets verified, promote it: fact → the matching
  kb sheet, project gotcha → docs/LEARNINGS.md, then note the promotion.
